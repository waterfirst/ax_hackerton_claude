# 한국투자증권 모의투자 × AI 오케스트레이션 투자 실험

이 폴더는 **한국투자증권(KIS) OpenAPI 모의투자**를 AI 에이전트로 운용하면서
**모든 매수/매도의 이유와 결과를 기록해 복기**하는 방법을 정리한 것이다.

> ⚠️ 투자자문이 아니라 **연구·교육·기록 목적**이다. 투자 책임은 본인에게 있다.
> ⚠️ APP KEY / SECRET / 계좌번호 / 토큰은 **절대 커밋·공개하지 않는다.**

---

## 1. 한국투자증권 모의투자 신청 방법

### 1-1. 계정·계좌
1. 한국투자증권 계좌 개설 (앱 또는 영업점).
2. [KIS Developers](https://apiportal.koreainvestment.com) 접속 → 로그인.

### 1-2. 모의투자 신청
1. KIS Developers → **모의투자** 메뉴 신청.
2. 모의투자 계좌가 생성된다 (계좌번호 앞 8자리 확인).
3. 기본 예수금 **1억원**이 지급된다.
4. 투자기간이 설정된다 (예: 3개월 단위, 만료 시 재신청).

### 1-3. API 키 발급
1. KIS Developers → **My App** → 앱 등록.
2. **모의(paper)용 APP KEY / APP SECRET** 발급.
3. (선택) 실전용 APP KEY / SECRET도 별도 발급 — 실전은 나중 단계.

### 1-4. 매매 가능 시간 / 체결 방식 (중요)
- **평일(영업일)만** 주문 접수·체결. 주말·공휴일 휴장.
- 매매시간: 실제 시장과 동일 (대략 평일 08:00~18:00, 정규장 09:00~15:30 + 시간외).
- **모의 체결 = 실제 시장에서 그 가격에 체결될 조건이 충족될 때만 체결.**
  단순 주문 입력만으로 즉시 체결되지 않는다 → 지정가 전략이 현실적이다.

## 2. 키 저장 (안전)

키는 **로컬 설정 파일에만** 둔다. 예: `~/.KIS/config/kis_devlp.yaml`

- 템플릿: [`kis_devlp.yaml.example`](./kis_devlp.yaml.example) 복사 후 값 입력.
- **Git 커밋 금지** (`.gitignore`에 추가), 공개 업로드·로그 출력 금지.
- 엔진은 이 yaml에서만 키를 읽는다 (코드에 하드코딩 금지).

잔고조회 최소 코드 흐름:
```
POST {vps}/oauth2/tokenP           # 토큰 발급 (1분당 1회 제한 → 파일 캐시)
GET  {vps}/uapi/domestic-stock/v1/trading/inquire-balance
     header tr_id = VTTC8434R (모의 잔고)   # 실전은 TTTC8434R
```

## 3. AI 오케스트레이션 구조

투자 판단을 **한 AI에 맡기지 않고 역할을 나눈다.**

| 역할 | AI | 하는 일 |
|---|---|---|
| 전략 총괄·리스크 승인·복기 | **Claude** | 지시·점검·게이트 판정 (짧게, 토큰 절약) |
| 실행 엔진 | **Codex** | 코드 작성·주문 집행·백테스트·자동기록 |
| 리서치 | Gemini | 장전 뉴스·매크로 스캔 |
| 센티먼트 | Grok | 실시간 속보·심리 |
| 리포트 | Z AI | 일일/주간 요약 |

소통은 마크다운 파일 핑퐁(`orch/PROTOCOL.md`, `TASKS.md`, `REPORT.md`)으로 한다.

### 예측 모델
- KOSPI **다이오드 회로 예측모델**(이 저장소의 `kospi_diode_mcp`)이 시가/종가/레짐 신호를 공급.
- 신호 → 방향 + 확신도 등급(강/중/약) → 가드레일 검증 → 지정가 모의주문.

## 4. 일일 파이프라인

```
07:35 시가예측 ─┐
12:35 종가예측 ─┼→ 신호(방향/등급/이유) → 가드레일 → 모의 지정가 주문 → trade_log 기록
장중 레짐 체크 ─┘
16:35 채점 → 마감 복기(성과·적중·틀린이유) → 모델 미세조정
```

리스크 하드룰(예): 단일종목 ≤15%, 1회주문 ≤500만, 일일손실 -2% 킬스위치, 지정가만, 물타기 금지, 손익비 ≥1.5 지향.

## 5. 매매기록 & 복기 체계 (핵심)

**"왜 샀는가 / 왜 안 샀는가 / 왜 팔았는가"를 반드시 남긴다.** 이게 모델 개선의 연료다.

- [`trade_log.csv`](./trade_log.csv) — 모든 주문/체결 기록 (이유·확신도 포함).
- [`decision_log.md`](./decision_log.md) — 판단 서술 기록 (진입/보류/청산 이유). **"아무것도 안 한 이유"도 기록.**
- [`records/`](./records/) — 일자별 복기 (`YYYY-MM-DD.md`). 템플릿: [`daily_review_template.md`](./daily_review_template.md).

측정 지표: 누적수익률 · MDD · 승률 · **손익비** · vs코스피(buy&hold) · 신호tier별 실현수익.

## 6. 단계별 접근 (자본 보존 우선)

1. **모의 조회 확인** → 토큰·잔고·주문가능금액 OK.
2. **소액 규칙 모드** — 1억이라도 초기엔 전액 투입 금지, 30~40%만.
3. **정식 모의운용** — 예측+리스크규칙 결합 주문, 매일 복기.
4. **검증 후 실전 소액** — 지표가 재현되면(기대값>0·MDD통제·손익비≥1.5) 실전 소액 보조.

핵심은 대박이 아니라 **재현 가능한 규칙**: 맞을 때 크게, 틀릴 때 작게, 애매하면 쉰다.
