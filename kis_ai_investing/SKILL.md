# AI + KIS Investing Skill

## 목적
이 스킬은 AI 에이전트가 **한국투자증권 OpenAPI 기반 투자 실험**을 다룰 때
안전하고 재현 가능하게 행동하도록 돕는다.

---

## 핵심 역할

AI는 다음 순서로 행동한다:

1. **계좌/API 연결 상태 확인**
2. **데이터 수집**
3. **모델 신호 계산**
4. **매수/매도/보류 판단 초안**
5. **리스크 규칙 적용**
6. **기록 저장**
7. **복기 작성**

---

## 안전 규칙

### 1. 자격증명
- API 키, 시크릿, 토큰, 계좌번호는 절대 출력하지 않는다
- 키는 로컬 설정 파일에만 둔다
- Git에 커밋하지 않는다

### 2. 실전 주문
- 사용자의 명시적 승인 없이 실전 주문하지 않는다
- 가능하면 먼저 모의투자 또는 shadow trading을 사용한다

### 3. 손실 통제
- 최대 손실 허용 기준이 없는 전략은 실행하지 않는다
- 포지션 비중 제한이 없으면 먼저 제한을 만든다

### 4. 기록
- 모든 판단은 기록한다
- 특히 “아무 것도 안 한 이유”도 남긴다

## 추천 작업 순서

### Phase 1. 연결 점검
- 토큰 발급 확인
- 잔고조회 확인
- 시세조회 확인
- 주문가능금액 확인

### Phase 2. Shadow Mode
- 실전계좌 조회만
- 실제 주문 없이 가상체결
- 예측 모델 성과 기록

### Phase 3. Mock Mode
- 모의계좌로 주문/체결 테스트
- 비중 제한하에 전략 검증

### Phase 4. Live Assist
- 실전은 인간 승인 후 보조만 수행

## 판단 프레임

AI는 매일 아래 질문에 답해야 한다.

1. 오늘 시장은 정상/위기/과열/횡보 중 무엇인가
2. 시가 모델은 무엇을 말하는가
3. 종가 모델은 무엇을 말하는가
4. 반도체/수급/매크로는 같은 방향인가
5. 매매를 해야 하는가, 쉬어야 하는가

## 출력 형식 권장

### 일일 프리마켓
- 시장 레짐
- 핵심 변수 3~5개
- 오늘 기본 시나리오
- 금지 시나리오
- BUY / SELL / HOLD 초안

### 장마감 후
- 예측 vs 실제
- 체결 가정 손익
- 잘한 점
- 틀린 점
- 수정 포인트

## 수익률 최대화 원칙

1. 맞을 때 크게, 틀릴 때 작게
2. 애매한 날 쉬기
3. 위기장에서는 공격보다 방어
4. 동일 논리 반복 진입 제한
5. 예측 정확도보다 기대값을 본다
